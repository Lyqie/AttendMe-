AttendMe

基于云端多模态大模型的实时注意力检测系统

期末项目报告 2026 年 6 月 Python · PySide6 · DashScope Qwen VL

项目概述

AttendMe 是一款 Windows 桌面注意力监测工具。它以固定间隔捕获用户屏幕截图， 通过云端多模态视觉大模型（DashScope Qwen VL）分析当前活动内容，实时给出 0–100 的专注度评分，帮助用户量化并改善数字工作习惯。

设计理念

云端 AI 分析 + 本地轻量 UI。所有计算密集的视觉理解任务卸载到云端， 本地仅运行一个轻量级 PySide6 悬浮窗，负责截屏捕获、结果展示与数据记录。 用户无需高性能 GPU，一台普通办公电脑即可使用。

核心功能

AI 评分

云端多模态专注度评分

系统将屏幕截图编码为 base64 JPEG，调用 DashScope Qwen VL 模型进行分析。 System Prompt 将人类活动划分为三个专注层级：

80–100
流动 Flow
编程、写作、设计、数据分析、阅读论文
40–79
浮动 Drifting
邮件、网页浏览、会议、项目管理、工作聊天
0–39
分散 Scattered
社交媒体、视频、游戏、空闲桌面、股票交易

模型返回 JSON 结构化结果，包含 score / category / activity / reasoning 四个字段， 支持代码块包裹、尾部逗号等常见格式变体的容错解析。

悬浮窗

悬浮窗 —— 折叠态

190×260px 无边框置顶窗口，通过 Windows DWM API 实现毛玻璃（Acrylic/亚克力）背景效果。 核心视觉元素为 QPainter 手绘的环形分数进度条： SVG 弧线随分数变化以动画过渡，外圈带有呼吸式脉冲光晕。 同时显示当前活动窗口标题及两个操作按钮（临时忽略 / 展开详情）。 窗口可自由拖拽，右键弹出上下文菜单。

仪表盘

展开面板 —— 仪表盘态

520×620px 详细仪表盘，包含以下子组件：

  • 环形分数：与悬浮窗同步的专注度可视化
  • 统计面板：今日专注时长、平均分、打断次数
  • 历史趋势图：平滑折线图 + 渐变面积填充，含高/中/低三档区域色带，支持 5 / 15 / 60 分钟时间范围切换
  • JSON 导出按钮：一键导出全部采样记录

图表使用 QPainter 纯手绘实现，无第三方图表库依赖。 包含点阵网格、坐标轴标注、渐变填充、悬停检测等完整图表功能。

智能

白名单系统

用户可配置进程名（如 code.exe / pycharm64.exe）或窗口标题为白名单。 命中白名单的应用自动获得 95 分，无需调用 API，节省配额并降低延迟。 支持通过 config.json 预配置，或运行时通过右键菜单动态添加。

暂离

临时忽略机制

用户可临时忽略当前窗口，默认持续 5 分钟（可在配置中调整）。 被忽略的窗口自动获得 80 分，同样无需调用 API。 过期后自动恢复检测。适用于短暂休息、查阅文档等场景。

数据

数据持久化与系统托盘

所有采样记录存入本地 SQLite 数据库（WAL 模式），包含时间戳、分数、 分类、活动描述、推理过程、进程名、窗口标题等字段。支持 JSON 格式一键导出。

最小化时收缩至系统托盘，图标为程序绘制的极光主题 "A" 字圆形图标。 托盘菜单提供显示悬浮窗、展开详情、退出应用等操作。

容错

启动健康检查与错误处理

启动时自动验证 API 连通性，获取可用模型列表并过滤出视觉语言模型。 若 API 不可达，弹出中文警告对话框。

运行时追踪连续推理错误次数：连续 3 次失败后自动停止截图定时器， 弹出严重错误对话框并提供重新启动监测的选项，防止静默失效。

核心技术实现

屏幕捕获与窗口感知

使用 mss 库进行跨平台快速截屏，支持多显示器配置。 截图按配置的最大尺寸缩放并转为 JPEG（质量 75%）以减少 API 传输负载。

通过 Windows user32.dll / kernel32.dll / psapi.dll 的 ctypes 调用获取前台窗口信息：

# 获取前台窗口句柄 → 进程 ID → 进程名 from ctypes import windll, byref, c_ulong hwnd = windll.user32.GetForegroundWindow() pid = c_ulong() windll.user32.GetWindowThreadProcessId(hwnd, byref(pid)) # → OpenProcess + GetModuleBaseNameW 获取进程名

云端视觉语言模型调用

API 客户端兼容 OpenAI Chat Completions 格式，支持 DashScope、OpenAI、vLLM 等 任意兼容端点。图片以 base64 data URI 形式嵌入请求体：

# 请求格式 (OpenAI 兼容) { "model": "qwen3-vl-flash", "messages": [{ "role": "user", "content": [ {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,..."}}, {"type": "text", "text": "<System Prompt>"} ] }] }

System Prompt 是评分准确性的关键，明确定义了三档专注层级的具体活动类型、 评分区间和判断标准，引导模型输出结构化 JSON。 解析函数具备容错能力：自动去除 Markdown 代码块包裹、修复尾部逗号、 并在 JSON 解析失败时回退到正则匹配提取关键字段。

PySide6 自绘 UI 系统

悬浮窗和仪表盘的核心视觉组件均在 QPainter 的 paintEvent 中手绘实现，不使用任何 UI 模板或 CSS 框架：

  • 环形进度条：QPainter 绘制 SVG 弧线路径，QVariantAnimation 驱动 400ms 平滑过渡，外圈叠加半透明光晕层配合 QTimer 实现呼吸脉冲
  • 颜色插值：分数 → 颜色的三阶光谱映射（玫瑰红 → 琥珀橙 → 薄荷绿），HSLA 色彩空间线性插值
  • 趋势图：QPainterPath 折线 + 渐变填充，点阵网格线，坐标轴标签，悬停 tooltip
  • 毛玻璃：调用 dwmapi.dll DWM 窗口合成 API 设置 Acrylic 亚克力背景

SQLite 数据层

数据库采用 WAL（Write-Ahead Logging）模式，提升并发读写性能。设计三张表：

表名用途关键字段
snapshots 核心采样记录 timestamp, score, category, activity, reasoning, process_name, window_title, ignored
whitelist 白名单配置 pattern (进程名/窗口标题), type (process/title)
ignore_list 临时忽略记录 process, window_title, expires_at

snapshots 表在 timestamp 建立索引以支持高效时间范围查询。 提供今日统计聚合（COUNT/AVG + CASE WHEN 分类）、应用分布 TOP 10、打断次数检测等查询。

多线程架构

推理任务在 QThread 中执行，通过 Qt 信号槽机制与主线程通信：

  • result_ready(dict) 信号 → 主线程更新 UI 并写入数据库
  • error_occurred(str) 信号 → 主线程累加错误计数并决定是否告警
  • 推理中收到新请求时自动跳过（busy 状态），避免请求堆积
  • 应用退出时通过 QThread.quit() / wait() 安全终止线程

config.json 是唯一的配置文件，分为五大区块：

区块说明关键参数

多 API 提供商支持

API 层采用 OpenAI 兼容格式，修改 api_base 和 model_name 即可切换至不同后端：

提供商api_basemodel_name 示例
DashScope (阿里云)https://dashscope.aliyuncs.com/compatible-mode/v1qwen3-vl-flash
OpenAIhttps://api.openai.com/v1gpt-4o
vLLM (自部署)http://your-server:8000/v1任意 VL 模型
Ollama (本地)http://localhost:11434/v1llava / minicpm-v

API Key 支持 config.json 直接填写或通过环境变量 DASHSCOPE_API_KEY 注入，避免密钥泄露至代码仓库。

总结与展望

项目亮点

  • 云端 + 本地协同：将计算密集的视觉理解卸载到云端，本地仅运行轻量 UI，降低硬件门槛
  • 精致的桌面 UI：纯 QPainter 手绘的环形进度条、趋势图、毛玻璃效果，设计完成度高
  • 可扩展的 API 层：OpenAI 兼容格式，支持 4 种以上不同后端，从云端 API 到本地模型均可接入
  • 容错设计：白名单/忽略机制减少 API 调用，连续错误检测防止静默失效，JSON 解析容错
  • 完整的 Nocturne 设计系统：从色彩、排版到动画，建立了可复用的设计语言

未来展望

  • 本地模型支持：集成 ONNX / OpenVINO 实现完全离线运行
  • 跨平台适配：macOS / Linux 兼容（替换 Win32 API 调用）
  • 数据洞察增强：周报/月报、生产力趋势分析、应用分类热力图
  • 智能提醒：分数持续偏低时弹出温和提醒，支持番茄钟模式
  • 隐私增强：敏感窗口自动暂停检测、本地推理开关
  • 数据同步：支持 WebDAV / S3 云端备份和历史数据跨设备同步
